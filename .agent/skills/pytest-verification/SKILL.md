# 파이썬 테스트 및 검증 스킬

## ✅ pytest 표준 및 Fixture 활용
- **프레임워크:** 파이썬 표준 `unittest` 대신 **`pytest`를 기본 프레임워크로 사용**합니다.
- **Fixture:** 반복되는 설정(Setup)이나 데이터 생성은 **`@pytest.fixture`로 정의**하여 테스트 코드의 중복을 제거합니다.
- **명명 규칙:** 테스트 파일은 `test_*.py`, 함수는 `test_*()` 형식을 엄수하여 **자동 탐지(Discovery)가 가능**하게 합니다.

## 🔄 테스트 실행 프로세스
- **선 계획 후 코딩:** 테스트 코드 작성 전, 검증할 시나리오와 기댓값을 **의사코드(Pseudocode)로 먼저 제안**하고 확인받습니다.
- **실행 안내:** 코드 제공 시 사용자가 즉시 실행해 볼 수 있도록 **`pytest <파일명>` 명령어를 함께 명시**합니다.
- **결과 가독성:** 테스트 실패 시 원인을 즉각 파악할 수 있도록 **`assert` 문에 상세한 에러 메시지**를 포함합니다.

## 🛡️ 검증 범위 및 무결성
- **포괄적 검증:** 정상 경로(Happy Path) 외에도 **경계값, 예외 처리(raises), 타입 오류 등 엣지 케이스**를 반드시 포함합니다.
- **코드 무결성:** 테스트 대상 모듈의 임포트부터 실제 검증 로직까지 **생략 없이 즉시 실행 가능한 전체 코드**를 제공합니다.
- **환경 독립성:** 테스트가 외부 환경(DB, API 등)에 의존하는 경우 `unittest.mock` 등을 활용해 **독립적인 테스트 환경**을 구성합니다.
